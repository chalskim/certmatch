4. CertMatch 플랫폼 개발 요구사항 명세서
(Software Requirements Specification, SRS)

1. 📌 문서 개요

1.1 목적
본 문서는 CertMatch 인증 매칭 플랫폼의 기능적·비기능적 요구사항을 정의하여, 개발팀, 기획팀, 디자인팀, 외부 파트너가 동일한 목표를 공유하고 제품을 일관성 있게 개발할 수 있도록 합니다.

1.2 범위
* B2B/B2C 하이브리드 인증 매칭 플랫폼
* 주요 인증: ISMS, ISMS-P, ISO 27001, CPPG, GS 등
* 핵심 사용자: 기업(의무/자율 인증 희망), 컨설턴트/심사기관, 교육기관, 심사원 지망생

1.3 목표 사용자 페르소나 (요약)
* 이지훈(38세, 스타트업 CTO): 빠르고 저렴하게 ISMS-P 인증을 원함
* 박수연(32세, 컨설턴트): 심사원 자격 취득 후 프로젝트 연결을 원함


2. 🎯 전체 시스템 목표
목표	설명
시간 절감	인증 준비 기간을 610개월 → 35개월로 단축
비용 절감	컨설팅 비용 40~50% 감소 (중개 수수료 포함)
매칭 정확도	AI 기반 매칭 성공률 ≥70%
정책 연계	KISA, TTA, KFQ 등 정부 데이터 실시간 연동
통합 경험	컨설팅 + 교육 + 채용 + 자동화 도구를 한 플랫폼에서 제공


3. 🧩 기능적 요구사항 (Functional Requirements)

3.1 사용자 관리 (FR-01)
ID	요구사항	우선순위
FR-01-01	사용자 유형별 회원가입 (기업, 컨설턴트, 교육기관, 개인)	High
FR-01-02	사업자등록번호/자격증 번호 기반 실명 인증	High
FR-01-03	OAuth 2.0 (네이버, 카카오, 구글) 로그인 지원	Medium
FR-01-04	프로필 작성/수정 (경력, 보유 자격증, 서비스 지역 등)	High

3.2 인증 정보 및 매칭 (FR-02)
ID	요구사항	우선순위
FR-02-01	인증 유형 선택 (ISMS, ISO 27001, CPPG, GS 등)	High
FR-02-02	예산, 지역, 희망 기간 입력 폼 제공	High
FR-02-03	AI 기반 파트너 추천 (컨설턴트/심사기관/교육기관)	Critical
FR-02-04	견적 요청 → 채팅 → 계약 체결 플로우 지원	High
FR-02-05	매칭 성공 시 수수료 결제 및 계약서 자동 생성	Medium

3.3 교육 및 채용 (FR-03)
ID	요구사항	우선순위
FR-03-01	심사원 교육, 보수교육, DIY 패키지 상품 등록/검색	High
FR-03-02	교육 이수 이력 프로필에 자동 반영	Medium
FR-03-03	정보보호 관련 채용공고 게시/지원 기능	Medium
FR-03-04	기업 프로필과 채용공고 연동	Low

3.4 정부 정책 연계 (FR-04)
ID	요구사항	우선순위
FR-04-01	KISA, TTA, KFQ 공식 API 연동 (공고 자동 수집)	Critical
FR-04-02	사용자 프로필 기반 정부 지원금/간편인증 자격 안내	High
FR-04-03	공고 알림 푸시/이메일 자동 발송	Medium

3.5 SaaS 자동화 도구 (FR-05)
ID	요구사항	우선순위
FR-05-01	Gap 분석 자동 생성 (ISO 27001/ISMS 기준)	High
FR-05-02	정책 문서 템플릿 자동 생성 (개인정보처리방침 등)	Medium
FR-05-03	위험 평가 체크리스트 제공 및 저장	Medium
FR-05-04	감사 준비 문서 패키지 다운로드	Low

3.6 리뷰 및 평판 시스템 (FR-06)
ID	요구사항	우선순위
FR-06-01	거래 완료 후 별점(1~5) 및 후기 작성	High
FR-06-02	평균 평점 및 후기 수 프로필에 표시	Medium
FR-06-03	악성 리뷰 필터링 (AI 기반 감정 분석)	Low


4. ⚙️ 비기능적 요구사항 (Non-Functional Requirements)
4.1 성능 (NFR-01)
요구사항	기준
플랫폼 응답 시간	≤1.5초 (95% 요청)
동시 사용자 수	최소 10,000명 동시 접속 지원
검색 결과 로딩	≤1초

4.2 보안 (NFR-02)
요구사항	기준
개인정보 보호	GDPR/PIPA 준수
데이터 암호화	TLS 1.3, AES-256
인증 보안	2FA 옵션 제공, 비밀번호 정책 강화
보안 감사	분기별 침투 테스트 수행

4.3 가용성 & 신뢰성 (NFR-03)
요구사항	기준
시스템 가용성	99.5% 이상 (월간 다운타임 ≤22분)
백업 주기	일일 자동 백업 + 실시간 로그 저장
장애 복구	RTO ≤30분, RPO ≤5분

4.4 확장성 (NFR-04)
요구사항	기준
모듈화 아키텍처	마이크로서비스 기반, 독립 배포 가능
인증 유형 확장	새로운 인증(예: ISO 45001) 추가 시 코드 변경 최소화
다국어 지원	한국어 기본, 영문/일문 확장 가능 구조

4.5 사용자 경험 (NFR-05)
요구사항	기준
접근성	WCAG 2.1 AA 준수
반응형 디자인	PC, 태블릿, 모바일 모두 지원
직관성	신규 사용자도 3클릭 내 핵심 기능 접근 가능


5. 🔌 외부 시스템 통합 요구사항
시스템	통합 내용	방식
KISA API	ISMS-P 간편인증 대상 여부, 공고 정보	REST API
TTA/KFQ	심사기관 목록, 자격증 정보	Open API 연동
PG사	결제 (신용카드, 계좌이체)	아임포트/토스페이 연동
알림 서비스	SMS, 이메일, 카카오 알림	알리고/메일건 연동
클라우드 인프라	AWS (EC2, RDS, S3, Lambda)	Infrastructure as Code (Terraform)

v1.5	AI 매칭 엔진, 정부 공고 연동, 리뷰 시스템	5개월

7. ✅ 검증 기준 (Acceptance Criteria)
*  페르소나 "이지훈"이 10분 내 ISMS-P 컨설턴트 3곳 비교 및 견적 요청 완료
*  페르소나 "박수연"이 심사원 교육 과정 검색 → 결제 → 이력 자동 반영 완료
*  KISA 공고 업데이트 시 1시간 이내 플랫폼에 반영
*  월 100건 매칭 시 수수료 자동 정산 및 정산 내역 제공

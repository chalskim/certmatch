CertLink 플랫폼 기능별 상세 요구사항서

📘 1. 시스템 개요
목적:기업, 인증심사기관, 컨설팅 회사, 교육기관을 연결하는 인증 통합 매칭 플랫폼 구축.범위:
* ISMS-P, GS, ISO 인증
* 인증 컨설팅 및 심사원 교육 매칭
* 공공 데이터 자동 연동

📗 2. 주요 기능 정의
구분	기능명	주요 내용	관련 모듈
1	사용자 회원관리	기업·컨설턴트·기관 구분 회원가입 및 로그인	Auth, DB
2	인증기관 조회	공공데이터 연동으로 최신 인증심사기관 목록 제공	API, DB
3	컨설팅 매칭	기업 요구조건에 맞는 컨설턴트 자동 추천	Matching Engine
4	교육과정 관리	인증심사원 교육과정 등록·수강·수료증 발급	Education
5	프로젝트 관리	인증 진행 단계별 진행상황 및 문서 관리	Project Tracker
6	공공데이터 연동	KISA·TTA·KFQ 등 API 자동 업데이트	Data Sync
7	평판 시스템	후기·평점·이력 기반 신뢰도 평가	Review
8	알림 서비스	일정, 교육, 심사 일정 자동 알림	Notification
9	관리자 대시보드	사용자·프로젝트 통계, 로그 관리	Admin Console
📙 3. 사용자별 기능 흐름
3.1 기업 사용자
단계	기능	입력	출력
1	회원가입	사업자등록번호, 담당자 정보	계정 생성
2	인증 종류 선택	ISMS-P / GS / ISO	준비 절차 안내
3	컨설팅 요청	예산, 일정, 필요 인증	매칭 결과
4	프로젝트 진행	심사 대응 문서 업로드	진행상황 표시
5	피드백 작성	후기 및 평점	컨설턴트 평가
3.2 컨설팅 업체 / 전문가
단계	기능	입력	출력
1	프로필 등록	전문분야, 이력, 인증경험	매칭 후보 등록
2	프로젝트 수락	제안서 제출	계약 체결
3	인증 지원	점검표, 대응전략	결과 보고서
3.3 인증기관 / 교육기관
단계	기능	입력	출력
1	기관 정보 등록	기관명, 인증 범위	기관 DB 반영
2	교육과정 등록	과정명, 일정, 강사정보	교육 리스트
3	수료 관리	수강생 이력, 평가	수료증 발급
📘 4. DB 구조 요약
테이블명	주요 필드	설명
User	user_id, type, name, email, password_hash, join_date	사용자 기본정보
Organization	org_id, name, type, location, contact	기관/기업 정보
Certification	cert_id, cert_type, level, valid_until	인증정보
ConsultantProfile	consultant_id, specialty, experience, rating	전문가 프로필
Project	project_id, company_id, consultant_id, status, start_date, end_date	인증 프로젝트
Education	edu_id, course_name, instructor, schedule, fee	교육과정 정보
Review	review_id, user_id, target_id, rating, comment	평판 및 후기
Notification	noti_id, user_id, message, type, read_flag	알림 관리
Log	log_id, action, user_id, timestamp	시스템 로그
📗 5. API 명세 요약
API 이름	메서드	경로	설명
회원가입	POST	/api/auth/register	사용자 등록
로그인	POST	/api/auth/login	JWT 기반 로그인
기관 목록 조회	GET	/api/institution/list	공공데이터 연동
컨설턴트 검색	GET	/api/consultant/search	조건 기반 검색
프로젝트 등록	POST	/api/project/create	기업 프로젝트 등록
교육 등록	POST	/api/education/register	교육기관 등록
후기 작성	POST	/api/review/add	평판 시스템 입력
알림 조회	GET	/api/notification/list	사용자 알림
데이터 동기화	PUT	/api/sync/publicdata	공공데이터 자동 업데이트
📘 6. 비기능 요구사항
구분	요구사항
보안	OAuth 2.0 / JWT 인증, HTTPS 암호화
확장성	MSA 기반 확장 구조, API Gateway 적용
성능	1000명 동시 접속 기준 응답속도 < 1s
가용성	Cloud 기반 이중화 (AWS/GCP)
데이터 보호	개인정보 암호화(AES256), 백업 주기 1일
호환성	RESTful API + OpenAPI 3.0 문서화

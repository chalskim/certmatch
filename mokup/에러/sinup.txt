1. 아이디 (ID) / 이메일 관련 오류
오류 상황
화면에 표시될 메시지 예시
추가 가이드
이미 사용 중인 아이디/이메일	이미 사용 중인 아이디입니다. <br> 이미 가입된 이메일 주소입니다.	- 실시간 중복 확인: 사용자가 입력을 마치고 포커스가 빠져나갈 때 즉시 결과를 알려주는 것이 좋습니다. <br> - 대안 제시: 다른 아이디를 입력해주세요. 와 같은 안내를 추가합니다.
올바르지 않은 형식	올바른 이메일 형식이 아닙니다. <br> 아이디는 영문, 숫자, 일부 특수문자만 사용 가능합니다.	- 입력 규칙 안내: 입력 필드 아래나 플레이스홀더에 허용 가능한 형식을 미리 알려줍니다. (예: user@example.com)
사용 불가 단어 포함	사용할 수 없는 단어가 포함되어 있습니다.	- 보안상 또는 운영상 금지된 단어(관리자, 운영자, 비속어 등)를 사전에 차단합니다.
길이 제한 위반	아이디는 5자 이상 20자 이하로 입력해주세요.	- 입력 필드에 maxlength 속성을 적용하여 물리적으로 더 이상 입력되지 않게 하는 것이 좋습니다.

2. 비밀번호 관련 오류
오류 상황
화면에 표시될 메시지 예시
추가 가이드
비밀번호 확인 불일치	비밀번호가 일치하지 않습니다.	- '비밀번호'와 '비밀번호 확인' 필드 모두에 오류를 표시해주는 것이 직관적입니다.
길이 기준 미충족	비밀번호는 최소 8자 이상이어야 합니다.	- 실시간 강도 표시: 비밀번호 입력 시 '약함', '보통', '강함' 등 시각적으로 안전도를 표시하면 사용자가 더 안전한 비밀번호를 설정하도록 유도할 수 있습니다.
조합 기준 미충족	영문 대/소문자, 숫자, 특수문자를 2가지 이상 조합해야 합니다.	- 어떤 조합이 충족되었는지 시각적으로 체크 표시를 해주면 사용자 경험이 크게 향상됩니다. <br> ☑ 영문 ☐ 숫자 ☐ 특수문자
아이디/개인정보와 동일	아이디나 이름과 동일한 비밀번호는 사용할 수 없습니다.	- 보안 강화를 위한 필수적인 검증 항목입니다.
너무 쉬운(취약한) 비밀번호	너무 쉬운 비밀번호는 사용할 수 없습니다. <br> (12345678, password 등)	- 자주 사용되는 취약한 비밀번호 목록을 기반으로 검증합니다.

닉네임 형식 오류	닉네임에는 특수문자를 사용할 수 없습니다.	- 허용/불가 문자를 명확히 안내합니다.
휴대폰 번호 중복	이미 가입된 휴대폰 번호입니다.	- 1인 1계정 원칙을 적용할 때 중요한 검증 항목입니다.
휴대폰 번호 형식 오류	올바른 휴대폰 번호를 입력해주세요. ('-' 없이 숫자만 입력)	- 자동으로 하이픈(-)을 제거하거나 포맷팅해주는 기능을 제공하면 사용자 편의성이 높아집니다.
